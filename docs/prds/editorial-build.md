# PRD: Editorial build — the render baseline, five ways (+ the fenced frontier)

**Ticket:** the first per-surface variant build (decision map "(fog) per-surface builds"; surface order of record: editorial → pdp → plp → checkout → a11y → how-it-was-built).
**Spec of record:** [ADR-0008](docs/adr/0008-store-surfaces-and-instrument.md) — `packages/reference/surfaces/editorial/` is the contract; the "What a variant may vary" section is the complete list of serialization freedoms. This PRD adds no design; it turns the spec into buildable slices.

## What this build is

The editorial surface — one article, prose plus exactly one interaction — served by the five core paradigms at `/{variant}/editorial/`:

| Slice | Variant prefix | Paradigm | Worker | Scaffold source |
|---|---|---|---|---|
| A | `vanilla` | static HTML, no runtime | `pm-vanilla` | none exists — recorded below |
| B | `react-next` | heavy hydration (Next.js) | `pm-react-next` | `create-next-app` + OpenNext `@opennextjs/cloudflare` (FINDINGS §adapters) |
| C | `astro` | islands | `pm-astro` | `create astro` (static output is adapter-free per FINDINGS; an island only if the cart interaction is idiomatically an island) |
| D | `qwik` | resumability | `pm-qwik` | `create qwik` + official `cloudflare-workers` adapter |
| E | `htmx` | hypermedia / server HTML | `pm-htmx` | no starter exists — recorded below (the pinned `htmx.org` npm package vendored into the variant's own assets, served same-origin — never a CDN include: the suite's request tracker fails any request off the composed origin) |
| F | `remix3` | fenced frontier (beta) | `pm-remix3` | hand-rolled Workers entry (ADR-0004 second addendum; spike is prior art) |

Editorial is the render baseline: the measured question is how much machinery prose needs. React/Next is the planning-time villain here — which is exactly why the page must be idiomatic, not sabotaged (ADR-0003 addendum: starter + published diff is the fairness mechanism).

## The contract every slice serves (from ADR-0008, restated not redesigned)

1. **Served DOM = the master**, modulo the serialization freedoms: attribute/class order, whitespace runs, comments, `script`/`style`/`link`/`template`, the `<head>` subtree, the chrome-slot subtree, and the variant's registered `PERMITTED_NOISE`. Everything else — elements, nesting, attributes and values, class names, text — byte-equal after normalization. The gate sees the JS-off SERVED document only.
2. **The shell skeleton**: skip link first, then `<div id="pm-chrome-slot"></div>` (variants only — the masters deliberately omit it; `packages/reference/test/reference.test.ts:102` asserts that), then `.pm-page` (masthead · main · `role="status"` live region · footer). Cross-surface links absolute to designated hosts, exactly as the master has them — including targets that 404 until their builds land (`/vanilla/pdp/…`, `/vanilla/checkout/`, `/vanilla/a11y/`, `/how-it-was-built/`). No suite assertion may require those targets to resolve yet.
3. **Canonical served cart state is EMPTY** (ADR-0008 §7): masthead count slot vacant at fixed width. Add to cart is per-paradigm client enhancement: update the `localStorage` cart (ADR-0004 §5 — `localStorage` holds the cart ONLY, and shared same-origin storage is how the cart survives a swap), populate the masthead count slot, announce through the `data-pm-status` region (WCAG 4.1.3). **No cart storage contract exists yet** — nothing anywhere names the key or value shape. Slice A defines and records it (key, value schema, count semantics, the status-announcement copy) as a committed contract every other variant and every later cart-bearing build (PDP, checkout, masthead everywhere) consumes — five independent inventions would break cart-survives-the-swap, silently. JS-off: the button is honestly inert; the page states nothing false (the master's copy already frames the button as "the only interactive element").
4. **Content = the committed per-snapshot essays** (`CRATE_ESSAY` / `FIXTURE_ESSAY`, `packages/reference/render/editorial.mjs`). Every precise number interpolates tray/manifest fields; the dateline is the manifest's freeze date; prose narrates allusively. Variants re-implement the DOM idiomatically; essay copy is content, and how a variant consumes it (build-time data import vs verbatim re-type) is slice A's call to record — under two constraints: no shared *component runtime* crosses variants (ADR-0003 §1), and the drift gate polices textual identity either way.
5. **Data access is the paradigm's real shape** (ADR-0002 §7): build-time variants (vanilla, astro-static) bake the snapshot trays in at build; request-time variants (react-next SSR, qwik, htmx, remix3) fetch trays through the edge Worker on the same plane. The build is snapshot-parameterized like `packages/reference/render/build.mjs` (`--snapshot fixture|crate`): CI builds/serves the fixture (CI never touches the crate), the deploy leg builds/serves the crate (trays are committed; image bytes stay R2-only). What the plane serves must match what `/api/snapshot` says it serves — the snapshot-aware suite (issue #11 pattern) is the check. **Turbo-cache hazard, concrete:** the origin job and the deploy job share the `turbo-origin-*` cache family, `run-local.mjs:108` and the deploy step both run `turbo run build` — an undeclared snapshot selector would replay the origin job's fixture-flavored dists straight onto the crate plane. Every snapshot-parameterized build task MUST declare the selector as turbo `env` and the tray files as `inputs`, following the `@pm/front#build` precedent (turbo.json documents exactly this failure mode for home's receipts).
6. **CSS its native way; fonts a controlled constant.** Each variant delivers the shared tokens/component/surface CSS by its own idiomatic delivery (ADR-0003 §2 — repackage-not-revalue; declared rules and rendered DOM are control, delivery is the measured variable). Fonts are NOT part of that freedom: ADR-0003 §8 holds them to identical files + identical loading everywhere — the canonical loading markup (`packages/tokens/fonts/loading-markup.html`: FamiljenGrotesk + PMCrateSymbols preloaded, PMWarnGlyph unpreloaded) ships verbatim, only the asset base path differing; files byte-identical to `@pm/tokens` (the existing font-leg suite pattern). Image URLs stay absolute `/assets/img/*` (the edge Worker serves them; no variant proxies or re-hosts images).
7. **The chrome is not the variant's job**: the front Worker injects the instrument into the slot and serves `measure.js` + chrome fonts from `/_pm/*`. A variant ships the slot div and nothing else chrome-shaped.

## Per-slice binding duties (the definition of done, per the handoff + ADRs)

Each variant slice, on landing:

- **Starter receipt** — scaffold from the framework's official starter; commit `variants/{name}/DIFF-TO-STARTER.md`: exact scaffold command, pinned starter/tool versions, every deviation from the starter output and why (expected entries: canonical markup, tokens import, composed-origin adapter, snapshot-parameterized data access). Verifiable by re-running the pinned command. For the two starterless paradigms the receipt records that fact and the minimal shape chosen: **vanilla** = hand-authored static pages + a build script in the `placeholder-static` mold (the whole tree is the diff, small enough to read); **htmx** = server-rendered HTML Worker + the pinned `htmx.org` npm package vendored into the variant's own assets and served same-origin as a script tag (htmx's documented install is a script tag, not a scaffold; a CDN include would fail the suite's same-origin request tracker).
- **Composition wiring** — new workspace under `variants/{name}` with `dev`/`build` scripts plus a package `deploy` script invoked via `pnpm --filter` in the CI deploy job (the placeholder precedent — no turbo `deploy` task exists; `run-local.mjs:48` spawns `pnpm run dev` per worker; `typecheck`/`test` where the workspace has a type/test surface; lint rides the root `//#lint` task, whose inputs already cover `variants/**`); service binding + `VARIANTS` entry in `workers/front` (front wrangler.jsonc + `src/index.js`); `tools/origin-suite/run-local.mjs` port + spawn entry; CI deploy step in `.github/workflows/ci.yml`; `workers/README.md` deploy notes. **Every variant self-serves under its `/{variant}/` prefix** — the front Worker forwards the ORIGINAL request and never rewrites paths, so route mapping, anchor hrefs, and asset URLs are the variant's job; framework base-path config is an expected DIFF-TO-STARTER entry. **Request-time variants additionally bind pm-edge themselves** (`services` in their own wrangler.jsonc — the front's EDGE binding doesn't reach a variant server-side; wrangler's local dev registry resolves it, and run-local already spawns edge), and the CI deploy step must deploy pm-edge BEFORE any variant that binds it (today edge deploys after the placeholders — reorder when B lands).
- **`SURFACE_CONTROLS` registration** — move the variant from `editorial.plannedVariants` to `editorial.variants`. The chrome then renders it as a live anchor and the "Served by N of M" note recounts from the arrays — nothing typed.
- **`PERMITTED_NOISE` registration** — the paradigm's idiomatic noise, each class declared: hydration markers/scoping hashes as inert residue; behavior attributes (`hx-*`, `on:*`, `q:*`) as their own declared class (they are the mechanism, not residue — ADR-0008 freedoms list; remix3's `rmx-target`/`rmx-src` mechanism attributes, if they appear in served DOM, are slice F's registration call per FINDINGS §2, not an ADR-0008-listed class). Vanilla registers nothing (`NO_NOISE` is the point). The registry entry is part of the diff-to-starter story.
- **Drift gate green** — a variant-vs-master comparison joins `drift.browser.test.ts` for `/{variant}/editorial/` (the masters-health block already proves the master side; this build adds the first real variant comparisons). Remix 3 is ADVISORY: warns, never fails (ADR-0003 first addendum) — the suite must encode that distinction, not skip the check.
- **Origin-suite page assertions** for every page the slice serves: canonical markup + chrome slot present; chrome injected with `data-pm-variant`/`data-pm-surface` correct and the serving variant `aria-current` in its own switcher row; fonts served byte-identical to `@pm/tokens` sources (the existing per-variant font-leg pattern); noise classes observed in the raw HTML for noisy paradigms (the `composed-origin.test.ts` placeholder-ssr precedent).
- **Verification discipline** — verify-slice workflow in the background + inline probes before the slice's single commit (explicit paths); push; CI green; merge is Rob's call.

**Slice A additionally owes** (first content-surface build, ADR-0008 §9 consequence):
- The **deployed-smoke re-render leg**: the post-deploy suite re-renders the editorial master from the RESOLVED snapshot (whatever `/api/snapshot` says the plane serves) and compares the served variant page against it — today's CI leg proves fixture-equivalence only. This is what makes "the plane serves the crate correctly" a checked claim, not a hoped one.
- The real-variant composition pattern the other slices copy (workspace layout, snapshot-parameterized build with the turbo env/inputs declaration, deploy step shape) — **including minting the snapshot selector itself**: no such env var exists anywhere yet. Name it, declare it as turbo `env` on every snapshot-parameterized build task, set it to `crate` on the deploy job's bare `pnpm exec turbo run build` step, leave `fixture` the default everywhere else, and thread it through `run-local.mjs`'s turbo build derived from `PM_SEED_DIR` — otherwise the documented local-crate verification run (`PM_SEED_DIR=tools/snapshot-capture/crate`) seeds a crate plane while build-time variants serve baked fixture pages, and run-local's "the one command holds either way" promise (lines 117–121) silently breaks.
- The **`NoiseSpec` behavior-attribute class**: the registry type carries only `attrPatterns`/`classPatterns` (`normalize.ts:48-53`), so ADR-0008's "behavior attributes get their own declared registry class" is currently inexpressible. Slice A extends the type (e.g. `behaviorAttrPatterns` — stripped identically, auditable as its own class) so D/E/F register mechanism attributes without touching shared drift-gate types mid-chain, keeping "B–F depend on A only" true.
- The **cart storage contract** (contract item 3): key, value schema, count semantics, announcement copy — committed where the shell contract lives, consumed verbatim by B–F and by the later PDP/checkout builds.

**Slice F additionally owes** (remix3-frontier decisions, re-verified at build time):
- Re-verify the beta pin (`3.0.0-beta.5` was newest as of 2026-07-11; the spike's `test.sh` is the canary on any bump); exact-pin sub-packages via the committed lockfile.
- Three-layer labeling, FINDINGS §7(c)'s enumeration exactly: (1) **on-surface plaque** — `data-pm-fenced="true"`, exact version, "excluded from every benchmark number" (final copy is this slice's job, the mechanism is spike-proven); (2) **chrome** — the switcher lists remix3 with a pre-release tag in the control itself, and the HUD on the remix3 variant shows RUM only (no lab snapshot exists by policy); (3) **receipts** — the bench runner never batches `/remix3/*`, turned from policy into MECHANISM: the runner refuses a remix3 variant id (§7(c)3 names the guard), with a test. The chrome today has no fenced-variant affordance for spine surfaces (the fenced pattern exists only as the PLP strategy row) — extending `SurfaceControls` for a fenced exhibit entry is this slice's one chrome touch, consistent with "fenced exhibits never get a column" (ADR-0005 §7 / ADR-0008 §3).
- Noise registration from FINDINGS §7(b)'s recorded list — plus `rmx-target`/`rmx-src` mechanism attributes if they appear in served DOM (FINDINGS §2; this slice's registration call).
- **Drift-normalizer extension, scoped**: the plaque is an element-level divergence from the master that `PERMITTED_NOISE` cannot excuse (it strips only attribute names and class tokens) — extend the normalizer and the pixel-leg neutralize step to drop `[data-pm-fenced]` subtrees **only in the fenced variant's own comparison** (an unconditional drop would let any core variant hide divergent DOM from the gate by marking it fenced); the origin suite asserts core-variant editorial pages carry NO `[data-pm-fenced]` element. With the scoped drop, the advisory comparison is green-by-default and a warning means REAL drift. (The PDP build's live-origin plaque is IN the PDP master — contract, not a drop.)
- **The two FINDINGS §8 hand-offs** this build must not inherit silently: prefix-mounting under the composed origin (route mapping, frame `src`/`rmx-src` generation, anchor hrefs, and asset URLs all prefix-aware at `/remix3/editorial/` — "this exhibit's largest unexercised seam") and committed automated browser coverage of the §5 behaviors (frame reload, `run()` anchor interception, history) in the gate wiring — `test.sh` covers only the HTTP-observable side.

## Slice order and dependencies

A (vanilla) → B (react-next) → C (astro) → D (qwik) → E (htmx) → F (remix3).
B–F each depend on A only (the pattern + the editorial gate wiring). One slice per session; a session may take two adjacent slices if the first lands with budget to spare — F always runs alone (beta re-verification is its own risk). **Slice E additionally owes the home editorial-row flip**: ADR-0007 §4 records that catalogue rows update one at a time *as surfaces land* ("a one-word edit per the design"), and E completes the surface — the flip is E's last act, distinct from the publication-time tense/verdict flips which stay out of this build.

## Non-goals (fences that hold)

- **No other surfaces** — PDP/PLP/checkout/a11y/how-it-was-built are later builds; masthead/footer links to them 404 by design until then.
- **No benchmark publication** — bench batches, lab bundles (`/_pm/lab/{surface}.json`), the chrome-constant re-measure (ADR-0001 addendum F), and the publication-time tense/verdict flips on home are the next arc step. Nothing in this build writes a lab receipt or a verdict. (Home's editorial ROW flip is the exception, owned by slice E per ADR-0007 §4 — a landed-surface fact, not a verdict.)
- **No spec-layer redesign** — masters, chrome anatomy, copy, and SURFACE_CONTROLS semantics are ADR-0008's; a slice that thinks the spec is wrong records an ADR addendum question, it doesn't improvise.
- **No placeholder retirement** — `sample` + the two placeholders keep serving the composed-origin proofs until a deliberate cleanup ticket; this build only adds.
- **No live Discogs calls** anywhere in this build (the live-origin demo is a PDP-build concern, and fenced).

## Acceptance criteria (the build, overall)

1. `/{v}/editorial/` serves for all five core prefixes + `/remix3/editorial/`, through the composed origin, locally (fixture) and deployed (crate).
2. The drift gate proves every core variant against the editorial master across all three profiles; remix3 compared in advisory mode.
3. `SURFACE_CONTROLS.editorial.variants` lists the five; `plannedVariants` is empty or gone; the chrome's reading table shows five live columns; each serving page's chrome marks itself `aria-current`.
4. Every variant's noise is registered and observed; vanilla stays `NO_NOISE`.
5. Five `DIFF-TO-STARTER.md` receipts + the remix3 labeling plaque exist.
6. The deployed smoke re-renders the editorial master from the resolved snapshot and compares the served pages (slice A leg), green on the plane.
7. CI green on every slice; nothing merges that turns the post-deploy smoke red on main (this PRD's own standing rule, inherited by every slice).
